
##Reflection

Clear and full documentation makes it easier not only for engineers and developers, but also for other members of cross-functional teams to pick up and understand code they didn't write themselves. Diagrams can help visualise architecture and flow, reducing the time a team member spends reading through unfamiliar code. Automated testing can add a level of safety, ensuring the code does not break. Version control with Git also helps team members stay in the loop, as it tracks history and context, enabling them to fix or revert any changes. Pair and mob programming spread knowledge across the team in real time, while peer reviews can prevent trouble down the road by catching issues early on, and ensure that more than one person is on top of each change. Together, these practices help build shared ownership and foster collaboration.
