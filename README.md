
##Reflection

Clear documentation makes it easier for team members of cross-functional teams to pick up and understand code written by others. Diagrams help visualise architecture and flow, reducing the time spent reading through unfamiliar code. Automated testing adds layers of safety, ensuring the code does not break. Version control with Git also helps team members stay on top of history and context, enabling them to revert to any previous version. Pair programming spreads knowledge across the team in real time, while peer reviews can prevent issues by catching them early on, and ensure that more than one person is on top of each change. Together, these practices help build shared ownership.
